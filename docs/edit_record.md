# 記録編集

## 基本事項
- 主に編集するのは composables の内容である
- Games.ts
  - ゲーム名、機体名、カラーコードなどゲームに関する基本情報を記載している
  - あまり変更を加えることはないが新作リリース時に修正する必要がある
  - ショット名などはi18n/locales内の翻訳ファイルen.ymlおよびja.ymlに記載する
- NewScoreRecords.ts
  - 最近の記録に表示する情報を記載している
  - 配列のデータ形式はScoreRecords.tsに記載している
- ReleaseNotes.ts
  - 更新履歴に関する情報を記載している
  - 他と異なり翻訳ファイルを活用する必然性があまりないのでプレイヤー名や更新内容を直接記述しても良い
- ScoreRecords.ts
  - 膨大な量のスコアデータをここで管理している
  - 配列データは { player { game: { shot type: { score, status, date, replay, detail } }という構造になっておりプレイヤーごとにデータを管理している
  - 既存の部門のスコア更新の場合はscoreからdetailの部分を更新する
  - 新規プレイヤーの場合はplayerを、既存プレイヤーの新規部門の場合はgameやshot type以降の項目を追加し、必要な情報を記述する
  - プレイヤーの表記順は原則的に五十音順。並び順がごちゃごちゃだとわかりづらいのでなるべく整理する
  - replayは一意である必要があり、replayの重複があるとエラーとなる
  - プレイ日時はSilenseleneの表記と必ずしも一致しない。何故ならSilenseleneは欧州の時間で表示してあり時差があるから
  - replayファイルはpublic/replays/ゲーム名/内にアップロードし、リプレイネームは既存のリプレイと重複しないものにする
